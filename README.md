# Project Overview

**한 줄 소개**: 전세 계약 정보를 입력하면 AI가 '집주인의 의도적 사기'와 '시장 가격 변동'이라는 서로 다른 두 위험을 각각 계량화하고, 9분면 매트릭스로 종합해 계약 전 의사결정을 돕는 예방형 전세사기 탐지 서비스.

|---|---|

# 1. Problem — 왜 전세사기는 줄지 않는가

## 1-1. 사회적 재난이 된 범죄

[📷 전세 사기 피해는 최근 몇 년 사이 단순 범죄를 넘어 사회적 재난으로 진화 — 피해자 3.4만 명 / HUG 3년 누적 적자 7조 그래프]

2024년 기준 누적 전세사기 피해자는 약 3.4만 명을 돌파했고, HUG(주택도시보증공사)의 보증사고 금액은 2021년 5,790억 원에서 2024년 44,896억 원으로 4년 만에 약 8배 폭증했습니다. 국토부가 추가로 인정한 피해 건수만 540건, 피해주택 매입 누적 5,889건 — 전세사기는 더 이상 개별 사건이 아니라 광범위한 사회 구성원에게 영향을 미치는 구조적 위험으로 확대되고 있습니다.

## 1-2. 특정 지역·특정 주거유형에 집중되는 패턴

[📷 서울 다세대주택 전세사기 피해 톱1 강서구 화곡동 — 피해건수/보증금액 테이블]

전체 건물 유형 중 다세대(빌라)는 피해 비중이 가장 높은 30%대를 차지했고, 서울 25개 구 중 강서구가 피해건수 916건·보증금액 1,889억 원으로 압도적 1위였습니다. 저희는 여기서 첫 가설을 세웠습니다 — **"전세사기는 무작위로 발생하지 않고, 특정 조건에서 반복되는 패턴이 있을 것이다."** 그래서 피해가 가장 집중된 강서구 화곡동을 분석 타깃 지역으로 확정했습니다.

## 1-3. 기존 정책은 왜 근본적 예방이 안 되는가

[📷 현존하는 전세사기 방지 정책(LTV 80%룰·등기부등본·전세보증보험) vs 2021-2024 임차권 등기명령 건수 5배 급증 그래프]

LTV 80% 룰, 등기부등본 확인, 전세보증보험이라는 세 겹의 방어선이 이미 존재합니다. 그런데도 임차권 등기명령(보증금을 못 돌려받아 법적 조치에 들어간 건수)은 2021년 대비 2024년 약 5배 급증했습니다.

원인을 뜯어보니 세 정책 모두 같은 구조적 결함을 공유하고 있었습니다.

| 정책 | 한계 |
|---|---|
| 등기부등본 | 소유권·선순위 권리는 보여주지만 **가격 설계와 시장 리스크는 설명하지 못함** (사후 정보) |
| LTV(전세가율) | 단일 기준이라 **지역·주택유형·시장 국면에 따른 실제 위험 차이를 반영하지 못함** |
| 전세보증보험 | 가입 가능 여부만 알려줄 뿐 **왜 위험한지, 무엇을 바꾸면 안전해지는지는 설명하지 않음** (블랙박스) |

> 결국 사용자는 '위험하다/안전하다'는 단편적 판단만 받을 뿐, **왜 그런 결과가 나왔는지, 무엇을 바꾸면 위험을 낮출 수 있는지**에 대해서는 충분한 설명과 대안을 제공받지 못하고 있었습니다.

# 2. Research — 전세사기는 두 개의 서로 다른 위험이 결합된 것이다

## 2-1. 데이터로 확인한 '인위적 조작'의 흔적

[📷 공간적 전이 위험 분석(IV 테이블) / 주택 가격 결정요인 분석 테이블 — deposit_overhang IV 0.54, effective_LTV IV 0.47]

2023~2025년 서울 강서구 화곡동 전월세·경매 데이터를 분석한 결과, **보증금 초과액(deposit_overhang)**과 **전세가율(effective_LTV)**이 과거 경매 발생과 가장 강하게 연관된 변수(IV 0.54, 0.47)로 나타났습니다. 동시에 헤도닉 가격 모델에서는 **법인 매입 시 시세보다 약 23.6% 저렴하게 거래**되는 패턴이 통계적으로 유의했습니다(계수 -0.269, p<0.001).

> 법인 매입과 주요 가격 요인의 결합이 인위적 가격 구조(깡통매매)를 형성한다 — 즉 우연한 시장 흐름이 아니라 **의도가 개입된 구조적 설계 위험**의 흔적입니다.

## 2-2. 데이터로 확인한 '시장 변동'의 독립적 위험

[📷 기준금리 변화에 따른 화곡동 집값 변동 테이블 — +1%p당 -2.58%]

동시에, 사기 의도가 전혀 없는 정상적인 계약이라도 계약 이후 기준금리가 4%p 오르면 화곡동 집값 지수는 약 -10.32% 하락하는 것으로 나타났습니다. 이는 **경제적 지표·시간 흐름·시장 가치 변화에 따라 증폭되는 별개의 위험**입니다.

## 2-3. 화곡동에서 가장 극명하게 드러난 제도적 공백

[📷 화곡동 LTV 분포 히스토그램(65.9%가 LTV 80% 이상) / 기존 임계값 대비 최적 임계값 5.22배 적중률 그래프]

화곡동 주택의 **65.9%가 이미 LTV 80% 이상**이어서, "LTV 80% 이하면 안전"이라는 범용 기준 자체가 이 지역에서는 무의미했습니다. 반면 데이터 기반으로 최적 임계값을 재산정하자 고위험군 적중률이 **5.22배** 향상됐습니다. 게다가 화곡동의 연간 가격 변동성은 31.34%에 달해, 전세가율 70%조차 안심할 수 없는 위험군이었습니다.

> **즉, 전세사기는 ① 물리적 조건·공간적 입지·가격 구조에 의해 형성되는 사기 의도가 개입된 '구조적 설계 위험'과, ② 경제적 지표·시간적 흐름·시장 가치 변화에 따라 증폭되는 '시장 변동 위험'이 서로 결합되어 나타나는 복합적이고 입체적인 리스크였습니다.**

이 발견이 이후 모든 설계의 근거가 됩니다: **하나의 점수로 뭉뚱그리지 말고, 두 위험을 분리해서 각각 계량화한 뒤 교차 판단해야 한다.**

[📷 이중 방어 시스템 구조 — Track A(가격·공간·패턴 기반) + Track B(환경·변동 기반)]

저희는 전세사기를 '집주인의 의도적 사기'와 '시장 가격 변동' 두 관점에서 완벽히 분리해서 필터링하기로 결정했습니다.

| Track | 성격 | 핵심 기법 |

# 4. Development — Track A: 가치 검증 및 사기 패턴 탐지

## 4-1. 1단계 — 헤도닉 가격 모델로 '적정 매매가'부터 만든다

Track A의 첫 단추는 "이 집이 얼마여야 정상인가"를 데이터로 정의하는 것이었습니다. 이게 없으면 보증금이 비싼지 싼지 판단할 기준 자체가 없기 때문입니다.

```python
# VIF 반복 제거 + 단계적 전진 선택(AIC)으로 헤도닉 모델 변수 확정
# Intercept-only AIC: 3346.19 → 최종 모델 AIC: -1921 (18개 변수)
model_final = sm.OLS(y, X_final).fit()   # y = ln(거래금액)
```

**왜 이렇게 설계했는가**: 단순 평당가가 아니라 건축연령², 면적×층, 건축연령×매수자법인 같은 **교차항·비선형항**을 직접 설계해 넣었습니다. 그 결과 매수자가 법인일 경우 계수가 -0.269(약 23.6% 저렴)로 나왔는데, 이건 "노후 빌라를 법인이 사들여 시세보다 싸게 거래 기록을 남기는" 전세사기 전형적 패턴을 정량적으로 포착한 것입니다. 최종 R²=0.680, AIC=-1921, 500회 부트스트랩으로 계수 신뢰구간(95% CI가 0을 포함하지 않음)까지 검증해 "우연한 상관관계가 아니라 통계적으로 탄탄한 신호"임을 확인했습니다.

```
ln(거래금액) = 9.7390 - 0.0686·건축연령 + 0.0159·전용면적
             - 0.2689·매수자_법인 + 0.1058·매도자_법인 + ...
```

## 4-2. 2단계 — 잔차와 공간 패턴으로 '조직적 사기'를 구분한다

헤도닉 모델이 예측한 적정가와 실거래가의 **잔차(Residual)**, 그리고 Local Moran's I(공간적 이상치 지표)로 "이 매물이 주변과 비교해 얼마나 이질적인가"를 계산했습니다. Global Moran's I = 0.0471, z-score 10.68(p<0.000001)로, 위험 매물들이 무작위가 아니라 **공간적으로 군집**한다는 것을 통계적으로 확인했습니다.

## 4-3. 3단계 — 핵심 파생변수와 WOE/IV로 비선형 관계를 구간화

```python
def find_optimal_bins(df, feature, target, max_bins=5):
    dtree = DecisionTreeClassifier(
        max_leaf_nodes=max_bins,
        min_samples_leaf=max(100, len(data)//(max_bins*2)),
        random_state=42,

    dtree.fit(X, y)
    thresholds = dtree.tree_.threshold[dtree.tree_.threshold != -2]
    return np.sort(np.unique(np.concatenate([[-np.inf], thresholds, [np.inf]]))).tolist()
```

**왜 의사결정나무로 구간을 나눴는가**: `deposit_overhang`(보증금 초과액)이나 `effective_LTV`(전세가율)는 위험도와 선형 관계가 아닙니다. 로지스틱 회귀에 그대로 넣으면 비선형 신호를 놓칩니다. 그래서 DecisionTreeClassifier로 "경매 발생률이 단조적으로 변하는" 최적 절단점을 자동으로 찾은 뒤, WOE(Weight of Evidence)로 변환했습니다.

```python
stats['dist_good'] = (stats['good'] / total_good) + epsilon
stats['dist_bad']  = (stats['bad']  / total_bad)  + epsilon
stats['woe'] = np.log(stats['dist_good'] / stats['dist_bad'])
stats['iv']  = (stats['dist_good'] - stats['dist_bad']) * stats['woe']
```

결과: `deposit_overhang` IV=0.5429, `effective_LTV` IV=0.4701 — 신용평가 업계 기준(IV>0.3은 강한 예측력)을 상회하는 핵심 변수임을 확인했고, VIF 검증(최대 5.24)으로 다중공선성도 배제했습니다.

## 4-4. 4단계 — 경매 라벨링: "위험하다"의 기준을 직접 정의하다

지도학습을 하려면 "실제로 위험했던 매물"이 무엇인지 라벨이 있어야 하는데, 전세사기는 공식 라벨이 없습니다. 그래서 법원 경매 데이터를 Selenium으로 크롤링해 개시결정일자를 추출하고, 자체 기준으로 라벨을 만들었습니다.

```
라벨링 조건(AND): 임차권 등기(보증금 미반환 법적 증명) 
                 + HUG/HF 인수 조건(보증사고 대위변제)
                 + 대항력 있는 임차인(선순위 존재)

라벨링 공식: 개시결정일자 - 계약일자 ≤ 4년 → 위험(1)
             (전세 계약 2년 + 갱신 2년 고려)
```

# 5. Development — Track B: 정량적 부도 리스크 분석

Track A가 "이 집주인이 의도적으로 위험하게 설계했는가"를 본다면, Track B는 "설계는 정상이라도 앞으로 3~4년간 시장이 변하면 얼마나 위험해지는가"를 계산합니다. 저희 팀은 이 문제를 **Merton의 구조적 부도 모델**을 부동산에 적용해 풀었습니다.

## 5-1. 집값을 확률과정으로 모델링한다 — μ, σ 추정

```python
monthly["r_t"] = np.log(monthly["P"] / monthly["P"].shift(1))
mu_hat    = monthly["r_t"].mean() * 12          # 연율화 평균 성장률
sigma_hat = monthly["r_t"].std(ddof=1) * np.sqrt(12)   # 연율화 변동성
# 결과: mu_hat = -0.007366, sigma_hat = 0.313363
```

**왜 월별 중앙값으로 지수를 만들었는가**: 개별 거래 단위로 수익률을 계산하면 이상치(자전거래, 초고가 거래)에 크게 흔들립니다. 월별 median 가격지수 → 로그수익률로 변환한 뒤, 표본 수가 10건 미만인 달은 제거해 신뢰도가 낮은 달을 걸러냈습니다. 화곡동 집값의 연간 변동성(σ)은 31.3%로 상당히 높게 나왔는데, 이게 "화곡동은 LTV 70%도 위험군"이라는 결론의 근거가 됩니다.

## 5-2. μ를 현재 금리 환경으로 보정한다

```python
# 매매 수익률 ~ 금리 변화 회귀: r_t = α + β·Δrate_t + ε
beta_clip  = np.clip(beta_raw, -0.2, 0.0)   # 금리↑→가격↓ 관계만 허용
beta_final = 0.5 * beta_clip                 # 과신 방지 수축(shrinkage)
mu_adj = mu_hat + beta_final * (r_current - r_hist_avg)
mu_adj_clipped = np.clip(mu_adj, mu_hat - 0.10, mu_hat + 0.10)
# 결과: mu_hat(보정 전) -0.7366% → mu_adj(보정 후) -0.2734%
```

**왜 β를 클리핑하고 절반으로 수축시켰는가**: 과거 데이터로 추정한 금리 민감도(β)를 그대로 쓰면 회귀의 노이즈가 그대로 증폭되어 미래 예측이 과신될 위험이 있습니다. 이 프로젝트의 목적은 "정확한 가격 예측"이 아니라 "현실적인 위험 반영"이므로, 경제적으로 타당한 부호(금리 상승 시 가격 하락)만 허용하고 추정치를 보수적으로 절반만 반영했습니다. 이런 보수적 보정이 실전 리스크 모델에서 왜 필요한지 팀 내에서 직접 논의하고 파라미터를 정한 과정입니다.

## 5-3. 회수율(α)을 실제 경매 데이터에서 추정한다

```python
auc_sold["alpha"] = auc_sold["매각가"] / auc_sold["감정가"]
# alpha_mean=0.7545, alpha_p10=0.5877 (보수적 회수율)
ALPHA_USED = alpha_p10   # 서비스에는 보수적 값을 채택
```

경매가 발생했을 때 실제로 얼마나 건질 수 있는지(α)를 감정가 대비 낙찰가 비율로 직접 계산했습니다. 평균(0.75)이 아니라 **하위 10% 분위수(0.59)를 서비스 기본값으로 채택**한 것은, 사용자에게 "낙관적인 평균"이 아니라 "보수적으로 안전한 기준"을 보여줘야 한다는 팀의 판단이었습니다.

## 5-4. PD·LGD·EL을 폐형식(closed-form)으로 계산한다

```python
def pd_gbm_closed_form(V0, B, T, mu, sigma):
    # PD = P(V_T < B) = Φ( (ln(B/V0) - (mu - 0.5σ²)T) / (σ√T) )
    z = (np.log(B/V0) - (mu - 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    return norm_cdf(z)

def expected_loss_closed_form(V0, B, T, mu, sigma, alpha):
    # E[(B - α·V_T)^+] : 만기 시 집값이 α만큼만 회수될 때의 기대손실액
    K = B / alpha
    d = (np.log(K) - m) / s
    return B*norm_cdf(d) - alpha*np.exp(m + 0.5*s**2)*norm_cdf(d - s)
```

**왜 시뮬레이션이 아니라 폐형식 수식을 썼는가**: 몬테카를로 시뮬레이션으로도 PD/EL을 구할 수 있지만, 웹 서비스에서 사용자가 매물 정보를 입력할 때마다 실시간으로 응답해야 하므로 반복 시뮬레이션은 속도상 부담입니다. GBM 가정 하에서는 PD와 EL 모두 표준정규분포 CDF로 표현되는 폐형식 해가 존재한다는 것을 활용해, 웹 요청당 즉시 계산이 가능하도록 만들었습니다.

## 5-5. 적정 보증금(B*)을 역산한다 — 이분탐색

```python
def find_B_star_by_EL_closed_form(V0, T, mu, sigma, alpha, shock, EL_CAP, ...):
    # EL(B) 은 B에 대해 단조증가 → 이분탐색으로 EL(B*) = EL_CAP 인 B* 탐색
    while (high - low) > tol:
        mid = 0.5 * (low + high)
        el_mid = EL_of_B_closed_form(mid, V0, T, mu, sigma, alpha, shock)
        if el_mid > EL_CAP: high = mid
        else: low = mid
    return low
```

**왜 이 함수가 이 프로젝트의 핵심인가**: 여기까지의 모든 계산(PD, LGD, EL)은 "위험하다/아니다"를 알려줄 뿐입니다. 하지만 사용자가 실제로 협상 테이블에서 쓸 수 있는 것은 "그럼 보증금을 얼마로 낮춰야 안전한가"라는 숫자입니다. EL이 사용자가 감내 가능한 손실 한도(EL_CAP)를 넘지 않는 최대 보증금을 이분탐색으로 역산해, "보증금을 조정하세요 / 월세로 전환하세요 / 보증보험에 가입하세요"라는 **실전 행동 강령**으로 연결했습니다. 이게 기존 서비스(LTV 계산기, 보증보험 가입 여부 조회)와 결정적으로 다른 지점입니다 — 경고에서 그치지 않고 협상 가능한 금액을 제시합니다.

## 7-2. 실제 서비스에서 발견한 한계 — 정직하게 기록

포트폴리오를 준비하며 실제 생성된 PDF 리포트 샘플을 다시 열어보니, 주소가 "N/A"로 비어 있던 케이스에서 **적정 매매가 추정치(V0)가 0만원**으로 계산되고, 그 여파로 PD_3yr·PD_4yr·LGD·EL·권장 안전 보증금이 모두 0으로 출력되는 케이스가 있었습니다. 헤도닉 모델의 예측값(V0)이 실패하면 그 하위에 있는 Track B의 모든 계산이 함께 무너지는 **연쇄 실패(cascading failure)** 구조였던 것입니다.

이는 지오코딩(주소→위경도) 실패나 필수 입력값 누락 시 V0 산출 자체가 불가능해지는 엣지 케이스로, 프로덕션 단계였다면 (1) V0 산출 실패 시 사용자에게 명시적 경고를 노출하고 (2) 다운스트림 계산을 아예 차단하는 방어 로직이 필요했다는 것을 실제 결과물을 검토하며 확인했습니다. 화려한 지표 뒤에 숨어 있던 이 실패 케이스를 스스로 찾아낸 것이, 모델 성능 수치보다 더 크게 남은 배움이었습니다.
